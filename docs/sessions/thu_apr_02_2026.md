## Key decisions locked in this session

- No clustering — individual pothole points only
- No proximity radius — street name matching only
- All fields kept in hazards.json: id, serviceName, latitude, longitude, location, status, requestDate, description
- Route-first logic: OSRM route → extract street names → filter potholes by street name match
- Fuzzy street name matching: expand abbreviations, strip numbers/punctuation, contains/is-contained-by check, split on & for intersections
- Testing: user confirms manually first, then test written to lock behavior in
- GitHub Pages from root on main → fetch path is web/data/hazards.json
- Scope locked: show route + hazards on map, nothing else
- Root index.html is a redirect shim to web/index.html
- No debug.js
